RESTful web apps

Representational state transfer

What the hell is that?

Basically, using HTTP methods (verbs) and routes to request system access and manipulate resources

What the hell is that?

HTTP methods: GET, POST, PUT, DELETE

Routes: http://mywebsite.com/notes/1234

GET request to /notes/123 - get note w/ id 123

POST request to /notes (with {data}) - create a note with {data}

http://mywebsite.com/{resource}/{id}

http://mywebsite.com/{resource}/{id}/{nested_resource}/{id}

GET request to /users/23/notes - gets all the notes for user w/ id 23

GET requests use query strings like /search?search_term=Game+of+thrones

POST requests use body like:


{
  "title": "this is title",
  "body": "some body text stuff"
}

Testing your code

Automated test-driven development

Red, Green, Refactor

1. First write a test for the code you wish you had

2. Run it and watch it fail (red)

3. Write the code to make it pass (green)

4. Refactor the code if you can w/o breaking the test
